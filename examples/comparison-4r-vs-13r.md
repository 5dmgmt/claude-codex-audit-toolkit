# 4R scope cut vs 13R ALL PASS — 比較分析

## 何が違ったか

Workshop Course 1 (4 ラウンドで scope cut) と SIFT Phase C (13 ラウンドで ALL PASS) は、同じツールキット (五月雨防止 v2 + 5 fixes) で監査したのに、結末が逆になりました。

これは **どちらかが失敗** ではなく、**対象の構造的特性に応じた正しい判断** です。

## 対比表

| | Workshop Course 1 | SIFT Phase C |
|---|---|---|
| **検査対象** | 教材ランブック (15 phase × 16 要素 = 240 母数) | システム構造 (DB / auth / dev bypass / 受講者ジャーニー) |
| **R1 件数** | 40 件 | 多数 (詳細不明だが 20+) |
| **R3 件数** | 6 件 | 18 件 |
| **R4 件数** | 6 件 (停滞) | 6 件 (減衰継続) |
| **最終ラウンド** | R4 で scope cut | R13 で ALL PASS |
| **確定形** | v3.4 確定 + _review-notes.md に未対応 finding 転記 | ALL PASS 通過 |

## なぜ Workshop は 4R で停滞したか

- 教材ランブック = **書き物**。Codex の指摘は「もっと細かく書け」「別の角度から書け」が無限に続く構造
- すでに 800+ 行に膨張しており、これ以上書き足すと **受講者が読めない**
- 件数停滞 (6 → 6) + 過去方針への別角度指摘 = scope creep のサイン

## なぜ SIFT は 13R で PASS したか

- システム構造 = **コード**。Codex の指摘は「実装を直せ」で具体的、対処すれば消える
- dev bypass のような構造的機能を後から作り直す必要があり、R10-R12 を消費
- R7-R12 で着実に減衰していたので、R13 で ALL PASS は予測可能だった

## 判断ポイント

### 4R で stop すべきか継続すべきかの差

両方とも R3 → R4 で件数停滞 / 微減でしたが:

- **Workshop**: 停滞 = 構造的限界 (書き物の細分化に天井がある) → stop
- **SIFT**: 一時停滞 = 反映途中 (R5 でまた減少した) → continue

判断材料:
1. **指摘の中身**: Workshop は「角度を変えた言い換え」、SIFT は「具体的実装の修正」
2. **受講者 / 利用者目線**: Workshop はすでに読める範囲を超えていた、SIFT はまだ実装の隙があった
3. **件数推移の前後**: Workshop は R3-R4 が同数、SIFT は R3 → R4 → R5 で減衰継続

## 教訓

### 1. ラウンド数そのものは判断基準ではない

「3 ラウンド超えたら scope cut」は経験則 / 警戒目安に過ぎません。SIFT のように 13 ラウンドかけて PASS する構造もあります。

### 2. 対象の性質で目安が変わる

- **書き物 (ランブック / 教材 / docs)**: 3-5 ラウンドで scope cut することが多い
- **コード (実装 / システム)**: 5-15 ラウンド継続して PASS することが多い

### 3. 件数推移を「2 ラウンド連続」で見る

R3-R4 だけでは判断できません。R3 → R4 → R5 の 3 点で:
- 全部減少 → 継続
- 1 ラウンド停滞 → 様子見 (次ラウンドで減れば継続)
- 2 ラウンド連続停滞 → scope cut 検討

### 4. ランブックの行数も判断材料

ランブックが受講者の読める範囲を超えていたら、構造的に scope cut すべきタイミング。Codex は「もっと書け」と言い続けるが、書きすぎると逆効果。
