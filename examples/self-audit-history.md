R3 で件数増加した原因の分析として、各ラウンドの軸が変わっていることも記録:

| ラウンド | 監査軸の重点 |
|---|---|
| R1 | 公式仕様 / 内部整合 / リンク / PASS-FAIL 分離 / 文書品質 (基本軸) |

## 教訓

### 1. 件数推移は **同一ファイルでの同一監査軸** で比較する

ファイル数 / 監査軸が変わると件数は当然変わる。「総件数下降」は範囲を固定して測定する必要がある。本ツールキットの収束判定 (`docs/04`) も「直近 2-3 ラウンドの総件数が下降傾向」と書いているが、**範囲が固定されている前提**を明示するべき (`docs/04` 改善余地)。

### 2. 大きな構造変更 (構造再編 / 新規 docs 追加) の直後は件数が一時的に増える

ea0ee20 のような大規模構造再編の直後の R3 は、整合性チェックが大量発生するため、件数増加が **scope creep ではなく構造再編の必然的後始末** であることが多い。R4 で減衰するかが正念場。

### 3. ファイル別単調上昇は scope creep 予兆として個別に追跡

総件数だけでなく **各ファイルの推移** を見て、単調上昇しているファイルは scope creep 予兆として個別判定する。本リポでは docs/07/manual (5→7→9) が最も注意を要する。

### 4. 範囲拡大時は新規ファイル件数を別計測

R3 のように範囲拡大した場合、「新規ファイルの初回件数」と「旧ファイルの推移」を分けて計測する。両者を混ぜて総件数で見ると判断を誤る。
